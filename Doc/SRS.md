## 1. บทนำ (Introduction)

### 1.1 วัตถุประสงค์ (Purpose)
เอกสารฉบับนี้คือข้อกำหนดความต้องการซอฟต์แวร์ (Software Requirements Specification: SRS) ของแพลตฟอร์มเว็บไซต์ (Front-end Web Application) สำหรับร้านค้าส่ง **วงษ์หิรัญค้าส่ง โคราช** โดยจุดประสงค์หลักของระบบคือการนำเสนอภาพลักษณ์แบรนด์ แคตตาล็อกสินค้า และข้อมูลสาขา เพื่อสนับสนุนการทำการตลาดผ่านระบบเครื่องมือค้นหา (SEO) 

### 1.2 ขอบเขตของระบบ (Scope)
- นำเสนอหน้าข้อมูลครบถ้วนสำหรับผู้ใช้งานระดับลูกค้าทั่วไป (Guest User)
- เรนเดอร์แผนที่จำลองของเขตจังหวัดนครราชสีมา (Google Map API)
- เก็บและแสดงข้อมูลสินค้าในรูปแบบจำลอง (Mock JSON Structure)
- มีปุ่มลัดนำเข้าสู่แพลตฟอร์ม LINE อย่างราบรื่น

### 1.3 คำจำกัดความ (Definitions)
| คำศัพท์ | ความหมาย |
|---|---|
| SPA | Single Page Application – โครงสร้างเว็บแบบโหลดเนื้อหาลื่นไหลโดยไม่ต้องสลับเพจใหม่ |
| Mock Data | ข้อมูลจำลองที่ฝังลงในโปรแกรมเพื่อเลียนแบบฐานข้อมูลจริง |
| SEO | Search Engine Optimization – การปรับแต่งโครงสร้างเว็บให้สอดคล้องกับพฤติกรรมดัชนีของ Google |
| GA4 | Google Analytics 4 – ระบบติดตามสถิติผู้เข้าชมเว็บไซต์ |

## 4. ความต้องการเชิงฟังก์ชัน (Functional Requirements)

### FR-01: โครงสร้างและการนำทาง (Routing)
| รหัส | ความต้องการ |
|---|---|
| FR-01-1 | ระบบต้องสามารถสลับเนื้อหาเพจได้อย่างรวดเร็วโดยไม่ต้อง Refresh ผ่าน React Router |

## 5. ความต้องการที่ไม่ใช่เชิงฟังก์ชัน (Non-Functional Requirements)

### 5.1 ประสิทธิภาพ (Performance)
- เว็บไซต์ต้องรองรับเรื่อง **Core Web Vitals** โดยหน้าแรกพร้อมใช้งานภายใน 1.5 วินาที
- ไฟส์ภาพทั้งหมด ต้องรองรับขนาดที่เหมาะสมและเรียกใช้กระบวนการ Lazy Load
- Bundle Size (ขนาดรหัสระบบหลังติดตั้ง) ต้องถูกบีบอัดให้เล็กที่สุด 

### 5.2 SEO และ Analytics
- มีการติดตั้ง HTML Meta tags: `title`, `description`, `keywords` อย่างสมบูรณ์ครบถ้วน
- รองรับ **Open Graph (og:)** แสดงรูปเมื่อถูกนำไปลิงก์โพสต์ใน Facebook หรือ LINE แชท
- รองรับ **Schema Markup (JSON-LD)** เป็นโจทย์ให้ Google ทราบว่าเว็บนี้คือห้างร้าน Wholesale Store
- ติดตั้ง gtag.js ควบคู่ไปกับ Google Analytics เพื่อวัดผลพฤติกรรม

### 5.3 ความสามารถในการใช้งาน (Usability)
- **Responsive 100%**: รองรับ Mobile 320px ขึ้นไป พร้อม Tailwind Classes `md:`, `lg:` สำหรับปรับสัดส่วนจอยักษ์
- สกอร์คอนทราสต์ทางสายตายึดมาตรฐาน WCAG ระดับพื้นฐาน (AA)

## 6. ข้อจำกัดของระบบ (Constraints)

| ข้อจำกัด | รายละเอียด |
|---|---|
| จำลองข้อมูล | โครงสร้างข้อมูลไม่ได้ฝังระบบ Backend สินค้าที่ปรากฏเป็น Mock Data ต้องทำการ Build เพื่อเปลี่ยนสินค้าเพิ่มเติม |
| Google Maps API Key | หากปริมาณการโหลดหน้าแผนที่สูงลิ่วเกินขีดจำกัด (Quota Limit ของ Google Platform) แผนที่จะกลายเป็นลายน้ำ |

## 7. สถาปัตยกรรมของระบบ (System Architecture)

### 7.1 Technology Stack
| ชั้น | เทคโนโลยี | รายละเอียด |
|---|---|---|
| **Frontend Setup** | React 19 + TypeScript + Vite 6 | Core Framework รูปแบบ SPA |
| **Styling Tool** | Tailwind CSS 4 | Utility Class ช่วยตกแต่งดีไซน์รวดเร็ว |
| **Icons & Maps** | Lucide React + vis.gl maps | ส่งเสริม UI Component |

### 7.2 Deployment Workflow
```
Code Edit (VS Code) 
       ▼
Git Push / Commit (บน GitHub)
       ▼
Vercel Hooks ทำการตรวจจับเหตุการณ์ (Detection)
       ▼
รันคำสั่ง `vite build` สร้างไฟล์ HTML+Assests
       ▼
Deploy ออนไลน์ไปที่ WongHiran Domain
```
